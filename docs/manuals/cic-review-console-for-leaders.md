# CIC 대표 리뷰 콘솔 사용 안내

이 문서는 `프로젝트 등록 승인/반려/중복·폐기`를 담당하는 CIC 대표용 간단 안내서입니다.  
목표는 한 가지입니다. `PM이 등록한 프로젝트가 우리 조직의 실제 운영 대상인지 빠르게 판단하고, 필요한 경우 수정 요청을 남기는 것`입니다.

## 1. 이 페이지는 무엇인가요

`CIC 대표 리뷰 콘솔`은 PM이 포털에서 등록한 프로젝트를 CIC 대표가 확인하고 결정하는 화면입니다.

이 화면에서 CIC 대표는 아래 세 가지 결정을 내립니다.

- `승인`
  - 우리 조직의 실제 운영 프로젝트로 확정합니다.
- `수정 요청 후 반려`
  - 프로젝트 자체는 맞지만, 정보가 부족하거나 잘못되어 PM이 수정 후 다시 제출해야 합니다.
- `중복·폐기`
  - 같은 사업이 이미 등록되어 있거나 운영 대상으로 삼지 않을 제안이므로, 이번 등록 제안을 종료합니다.

중요:
- `중복·폐기`는 **중복 사업 제거 목적의 실제 폐기 결정**입니다.
- 다만 시스템 내부에서는 이력 추적을 위해 기록은 남습니다.
- 즉, 운영 대상으로는 제외되지만 기록 자체를 완전히 삭제하는 방식은 아닙니다.

## 2. 무엇을 자세히 보면 되나요

이 화면은 `PM이 프로젝트 수정 화면에 입력한 정보`를 거의 그대로 보여주는 구조입니다.  
즉 CIC 대표는 별도 요약본만 보는 것이 아니라, PM이 실제로 올린 원문을 읽고 판단하게 됩니다.

아래 항목을 순서대로 보시면 됩니다.

### 2-1. 기본 정보

먼저 프로젝트의 기본 식별 정보를 확인합니다.

- 프로젝트명
- 정식 계약명
- 발주기관
- 등록 조직(CIC)
- 담당 PM

이 단계에서 먼저 확인할 질문은 아래와 같습니다.

- 이 사업이 우리 CIC에서 다루는 사업이 맞는가?
- 담당 PM과 조직 배정이 맞는가?
- 이미 같은 사업이 다른 이름으로 등록되어 있지는 않은가?

### 2-2. 계약 및 운영 정보

다음으로 운영 기준이 맞는지 확인합니다.

- 사업 유형
- 사업 기간
- 정산 유형
- 정산 기준
- 통장 유형
- 자금 입력 방식

이 항목은 실제 운영 방식과 바로 연결되므로, 빈 값이나 잘못된 값이 있으면 `수정 요청 후 반려`를 주는 것이 맞습니다.

특히 아래는 꼭 확인해 주세요.

- 정산 유형과 정산 기준이 비어 있지 않은가
- 통장 유형이 실제 사업 운영 방식과 맞는가
- 자금 입력 방식이 `통장내역 업로드`인지 `직접 입력`인지 맞는가

### 2-3. 예산

예산 섹션은 단순 참고가 아니라, `사업 규모와 운영 가능성`을 판단하는 핵심 근거입니다.

확인 항목:

- 총사업비
- 매출부가세
- 총수익
- 지원금
- 입금 계획

이 숫자가 지나치게 비정상적이거나, 계약서/실제 사업 규모와 맞지 않으면 바로 승인하지 말고 PM에게 보완을 요청해 주세요.

### 2-4. 등록 인력

누가 이 사업을 실제로 운영하는지 확인합니다.

- 팀명
- 등록 인력
- 역할
- 참여율

특히 아래를 봐 주세요.

- 팀이 아예 비어 있지 않은가
- 핵심 담당자가 누락되지 않았는가
- 참여율/역할 구성이 상식적으로 맞는가

### 2-5. 목적 및 메모

이 사업을 왜 하는지, 어떤 조건이 붙는지 보는 구간입니다.

- 프로젝트 목적
- 참여 조건
- 상세 설명
- 비고

짧더라도 이 구간은 매우 중요합니다.  
이 설명을 읽어도 사업의 목적과 범위가 이해되지 않으면, 그대로 승인하지 않는 것이 맞습니다.

### 2-6. 첨부 계약서

계약서는 `보조자료`가 아니라 가능한 한 직접 확인하는 기준 자료입니다.

확인 항목:

- 파일명
- 업로드일
- 계약서 보기 링크
- AI/휴리스틱 요약
- 주의 사항
- 다음 행동

중요:
- AI 요약은 보조 수단일 뿐입니다.
- 스캔본이거나 텍스트 추출이 약하면 요약이 부정확할 수 있습니다.
- 계약서에서 핵심 정보가 추출되지 않았다고 나오면, 사람이 직접 계약서를 보고 판단해 주세요.

### 2-7. 리뷰 이력

이전에 어떤 결정이 있었는지 확인합니다.

- 접수일
- 요청자
- 리뷰 이력
- 반려 사유

이미 반려된 프로젝트라면, PM이 그 사유를 반영해 수정했는지 확인한 뒤 다시 판단하면 됩니다.

## 3. 어떤 버튼을 눌러야 하나요

### 승인

다음 조건이면 `승인`이 맞습니다.

- 우리 조직의 실제 운영 프로젝트가 맞다
- 기본 정보, 운영 정보, 예산, 인력 구성이 큰 틀에서 맞다
- 당장 운영을 시작해도 무리가 없다

### 수정 요청 후 반려

다음 조건이면 `수정 요청 후 반려`가 맞습니다.

- 프로젝트 자체는 맞지만 정보가 부족하다
- 정산 기준, 통장 유형, 기간, 예산, 팀 구성 등 핵심 값이 비어 있다
- 계약서와 입력 정보가 맞지 않는다
- PM이 조금만 보완하면 다시 검토 가능한 상태다

반려할 때는 사유를 구체적으로 남겨 주세요.

좋은 예:
- `정산 기준이 비어 있습니다. 공급가액/공급대가 기준을 다시 입력해 주세요.`
- `계약서상 발주기관과 입력값이 다릅니다. 계약서 기준으로 수정해 주세요.`

### 중복·폐기

다음 조건이면 `중복·폐기`가 맞습니다.

- 같은 사업이 이미 등록되어 있다
- 중복 등록 제안이다
- 운영 대상으로 삼지 않을 제안이다

이 결정은 `중복 사업 제거 목적의 폐기`로 이해하시면 됩니다.  
다만 시스템은 이력 추적을 위해 기록을 남깁니다.

## 4. 반려 후에는 어떻게 되나요

반려는 삭제가 아닙니다.

흐름은 아래와 같습니다.

1. CIC 대표가 `수정 요청 후 반려`
2. PM이 같은 프로젝트의 `프로젝트 수정` 화면으로 이동
3. 화면 상단에서 반려 사유 확인
4. 정보 보완 및 계약서 재업로드 가능
5. `수정 후 다시 제출`
6. 다시 CIC 대표 리뷰 큐로 복귀

즉, PM은 새 프로젝트를 다시 만드는 것이 아니라 `같은 프로젝트를 수정 후 재제출`합니다.

## 5. 강력 권장: PM과 1:1 미팅으로 맞춰 주세요

이 콘솔은 빠른 승인 도구이지만, 모든 문제를 화면만으로 완벽히 판단할 수는 없습니다.

특히 아래 경우에는 **가능하면 PM과 1:1 미팅을 권장드립니다.**

- 사업 목적과 범위가 화면만으로 명확하지 않을 때
- 예산 규모가 큰데 설명이 부족할 때
- 계약서와 입력 내용이 다르게 보일 때
- 인력 구성이나 역할이 비정상적으로 보일 때
- 같은 사업인지 중복인지 확신이 서지 않을 때

권장 이유:

- 문서만으로는 맥락이 빠질 수 있습니다
- PM이 왜 그렇게 입력했는지 직접 들으면 판단이 훨씬 빨라집니다
- 반려 횟수를 줄일 수 있습니다
- 승인 후 운영 중 생길 QA를 줄일 수 있습니다

즉, `정보가 모호한 건 화면에서 억지로 결론 내리기보다 PM과 1:1로 맞추는 것이 더 안전합니다.`

## 6. 자주 묻는 질문

### Q. 승인한 프로젝트도 다시 바꿀 수 있나요?

네. 승인 완료된 프로젝트도 다시 `반려` 또는 `중복·폐기`로 조정할 수 있습니다.

### Q. 반려하면 프로젝트가 삭제되나요?

아니요. 삭제되지 않습니다.  
프로젝트는 유지되고, PM이 같은 프로젝트를 수정해서 다시 제출합니다.

### Q. 중복·폐기는 진짜 폐기인가요?

네. 운영 판단 기준으로는 `이번 등록 제안을 폐기`하는 결정입니다.  
다만 시스템은 이력 관리를 위해 기록을 남깁니다.

### Q. 계약서는 어디서 확인하나요?

우측 상세 패널의 `첨부 계약서` 섹션에서 확인합니다.

### Q. AI 요약만 보고 결정해도 되나요?

권장하지 않습니다.  
AI/휴리스틱 요약은 참고용이고, 중요한 판단은 원문 계약서와 PM 입력 정보를 함께 보셔야 합니다.
