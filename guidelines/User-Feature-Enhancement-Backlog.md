# User Feature Enhancement Backlog (ServiceNow-grade Target)

## 목적
- 기술 안정성(P0)을 넘어서, 실제 사용자(관리자/재무/PM/감사)가 체감하는 생산성과 신뢰를 높이는 기능 후보를 우선순위로 정리한다.
- 각 항목은 "바로 구현 가능한 수준"으로 정의한다.

## P0 (즉시 착수)

1. 역할별 홈 대시보드
- 관리자/재무/PM/감사 별로 첫 화면 KPI와 액션 버튼 분리.
- 성공 기준: 로그인 후 2클릭 이내 핵심 업무 진입.

2. 결재 Inbox/Queue 통합
- 지출/증빙/인력변경 요청을 하나의 큐로 모아 상태/우선순위/마감 임박 필터 제공.
- 성공 기준: 승인 소요시간 20% 이상 단축.

3. 충돌 복구 UX
- `version_conflict` 발생 시 자동 재조회 + 차이점 표시 + 재시도 버튼.
- 성공 기준: 사용자 수동 새로고침 의존 제거.

4. 감사로그 검색/필터
- 행위자, 대상, 기간, 액션 유형으로 필터링 + CSV 내보내기.
- 성공 기준: 감사 질의 대응 시간 절반 이하.

5. 알림 센터
- 승인 필요/반려/정책 위반/증빙 누락을 앱 내 알림으로 통합.
- 성공 기준: 놓친 승인 건수 0건.

## P1 (다음 스프린트)

1. 동적 승인선(Approval Matrix)
- 금액/프로젝트/리스크 등급 기준으로 결재선 자동 분기.

2. 증빙 검증 보조
- 파일 누락/형식 불일치/금액 불일치 규칙 자동 체크.

3. 사용자 셀프서비스
- 권한 요청, 프로젝트 이동 요청, 계정정보 수정 요청을 표준 플로우화.

4. PM 전용 프로젝트 온보딩
- 프로젝트 등록 제안 시 체크리스트/필수 필드 검증/승인 상태 추적 제공.

5. 운영 캘린더
- 마감일, 정산 주기, 백업 점검일, 정책 변경일 캘린더화.

## P2 (제품 차별화)

1. SLA 보드
- 업무 유형별 목표 처리시간/실적/위반 원인 가시화.

2. 플레이북 자동 제안
- 반려 사유/감사 이슈 패턴 기반으로 다음 액션 템플릿 추천.

3. 멀티채널 알림
- 이메일/슬랙/카카오워크 등 외부 채널 연동.

4. 업무 자동화 룰 엔진
- "조건-행동" 규칙으로 반복 업무 자동 실행(예: 특정 금액 초과 시 자동 감사 태깅).

5. 업무 품질 점수
- 프로젝트별 증빙완성도/승인지연/오류재발률 점수화.

## 실행 원칙
- 먼저 `P0`는 API 안정성(인증/RBAC/페이징/충돌복구)과 함께 묶어 릴리즈한다.
- 기능은 "측정 가능한 지표" 없이 배포하지 않는다(처리시간, 재작업률, 누락률).
- UI 추가 전에 권한 모델과 감사 추적 가능성을 먼저 확정한다.
