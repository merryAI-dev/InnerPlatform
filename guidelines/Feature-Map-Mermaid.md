# Business Management Platform - Feature MAP (기능 중심)

아래 다이어그램은 개발 구현체보다 실제 업무 흐름(누가, 언제, 무엇을 처리하는지) 중심으로 정리한 맵입니다.

```mermaid
flowchart LR
  subgraph R0[사용자 역할]
    R1[관리자]
    R2[사업담당자 PM]
    R3[실무자]
  end

  subgraph J0[업무 여정]
    J1[1. 사업 등록/기획]
    J2[2. 예산 수립]
    J3[3. 인력 배치]
    J4[4. 집행: 지출/증빙]
    J5[5. 주간 캐시플로 점검]
    J6[6. 인건비 공지/인지]
    J7[7. 월간 정산 마감]
    J8[8. 감사/리포트]
  end

  subgraph A0[관리자 기능]
    A1[전사 대시보드]
    A2[프로젝트 관리\n목록/상세/등록]
    A3[예산/지출 관리]
    A4[승인 관리]
    A5[사용자/권한]
    A6[감사 로그/설정]
  end

  subgraph P0[포털 기능]
    P1[온보딩]
    P2[내 사업 현황]
    P3[지출/변경 요청 제출]
    P4[주간 캐시플로 입력]
    P5[인건비 공지 확인]
    P6[교육/경력]
    P7[신규 사업 제안]
  end

  subgraph C0[협업 접점]
    C1[전사 게시판 공지]
    C2[알림 패널]
    C3[반려/재요청 루프]
    C4[진행 상태 추적]
  end

  subgraph O0[핵심 산출물]
    O1[실행 예산]
    O2[지출 세트/증빙]
    O3[주간 현금흐름 현황]
    O4[인건비 상태]
    O5[월 마감 상태]
    O6[운영 리포트]
  end

  subgraph K0[기대 효과]
    K1[누락 감소]
    K2[마감 예측 가능성 향상]
    K3[감사 대응 시간 단축]
  end

  R1 --> A1
  R1 --> A2
  R1 --> A3
  R1 --> A4
  R1 --> A5
  R1 --> A6

  R2 --> P1
  R2 --> P2
  R2 --> P3
  R2 --> P4
  R2 --> P5
  R2 --> P7
  R3 --> P3
  R3 --> P6

  J1 --> J2 --> J3 --> J4 --> J5 --> J6 --> J7 --> J8

  A2 --> J1
  A3 --> J2
  A5 --> J3
  P3 --> J4
  P4 --> J5
  P5 --> J6
  A4 --> J7
  A6 --> J8

  C1 --> C2 --> C4
  C3 --> C2
  P3 --> C3
  A4 --> C3
  J4 --> C4
  J5 --> C4
  J6 --> C4
  J7 --> C4

  J2 --> O1
  J4 --> O2
  J5 --> O3
  J6 --> O4
  J7 --> O5
  J8 --> O6

  O1 --> K1
  O2 --> K1
  O3 --> K2
  O5 --> K2
  O6 --> K3
```

## 한 줄 요약
- 이 플랫폼은 `사업 등록 → 집행 → 주간 점검 → 인건비 → 월마감 → 감사`를 관리자/PM/실무자가 협업해 끊김 없이 처리하도록 설계되어 있습니다.
